    7. Deployment Guide¶

    This guide covers various deployment options for Cloud Native GIS.

    7.1 Docker Deployment (Recommended)¶

    7.1.1 Prerequisites¶

    • Docker 20.10+
    • Docker Compose 2.0+
    • Git

    7.1.2 Step-by-Step Installation¶

    # 1. Clone the repository
git clone https://github.com/kartoza/CloudNativeGIS
cd CloudNativeGIS

# 2. Initialize submodules
git submodule update --init --recursive

# 3. Copy configuration templates
cp deployment/.template.env deployment/.env
cp deployment/docker-compose.override.template deployment/docker-compose.override.yml

# 4. Edit environment variables
nano deployment/.env

# 5. Start the application
make up

    7.1.3 Docker Services¶

    Service Description Port
    nginx Reverse proxy 80
    django Application server 5000
    db PostgreSQL/PostGIS 5432
    dev Development server 5000

    7.1.4 Verifying Installation¶

    1. Open http://localhost/ in your browser
    2. Access admin at http://localhost/admin/
    3. Default credentials are in your .env file

    7.2 Production Deployment¶

    7.2.1 Using Docker Compose¶

    # docker-compose.prod.yml
version: '3.8'

services:
  nginx:
    image: kartoza/cloud_native_gis:latest
    restart: always
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./ssl:/etc/nginx/ssl:ro
    depends_on:
      - django

  django:
    image: kartoza/cloud_native_gis:latest
    restart: always
    env_file:
      - .env.prod
    depends_on:
      - db

  db:
    image: kartoza/postgis:15-3.3
    restart: always
    volumes:
      - pgdata:/var/lib/postgresql/data
    env_file:
      - .env.prod

volumes:
  pgdata:

    7.2.2 Environment Variables for Production¶

    # .env.prod
DEBUG=False
SECRET_KEY=your-very-secure-secret-key
ALLOWED_HOSTS=your-domain.com,www.your-domain.com
DATABASE_URL=postgis://user:password@db:5432/cloudnativegis
CSRF_TRUSTED_ORIGINS=https://your-domain.com

    7.2.3 SSL/TLS Configuration¶

    For HTTPS, use Let's Encrypt with certbot:

    # Install certbot
apt install certbot python3-certbot-nginx

# Obtain certificate
certbot --nginx -d your-domain.com

    7.3 Kubernetes Deployment¶

    7.3.1 Helm Chart (Coming Soon)¶

    A Helm chart for Kubernetes deployment is planned for future releases.

    7.3.2 Basic Kubernetes Manifests¶

    # deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: cloud-native-gis
spec:
  replicas: 3
  selector:
    matchLabels:
      app: cloud-native-gis
  template:
    metadata:
      labels:
        app: cloud-native-gis
    spec:
      containers:
      - name: django
        image: kartoza/cloud_native_gis:latest
        ports:
        - containerPort: 5000
        envFrom:
        - secretRef:
            name: cloud-native-gis-secrets

    7.4 Django Library Integration¶

    7.4.1 Installation¶

    pip install cloud-native-gis

    7.4.2 Django Settings¶

    # settings.py
INSTALLED_APPS = [
    'django.contrib.admin',
    'django.contrib.auth',
    'django.contrib.contenttypes',
    'django.contrib.sessions',
    'django.contrib.messages',
    'django.contrib.staticfiles',
    'django.contrib.gis',
    # Cloud Native GIS
    'cloud_native_gis',
    # Required dependencies
    'rest_framework',
    'rest_framework_gis',
    'corsheaders',
]

# Database with PostGIS
DATABASES = {
    'default': {
        'ENGINE': 'django.contrib.gis.db.backends.postgis',
        'NAME': 'your_database',
        'USER': 'your_user',
        'PASSWORD': 'your_password',
        'HOST': 'localhost',
        'PORT': '5432',
    }
}

    7.4.3 URL Configuration¶

    # urls.py
from django.urls import path, include

urlpatterns = [
    path('admin/', admin.site.urls),
    path('gis/', include('cloud_native_gis.urls')),
]

    7.4.4 Run Migrations¶

    python manage.py migrate

    7.5 Scaling¶

    7.5.1 Horizontal Scaling¶

    For high-traffic deployments:

    1. Use a load balancer (nginx, HAProxy, or cloud LB)
    2. Run multiple Django instances
    3. Use Redis for caching and Celery broker
    4. Consider a managed PostgreSQL service

    7.5.2 Performance Tuning¶

    # settings.py
CACHES = {
    'default': {
        'BACKEND': 'django.core.cache.backends.redis.RedisCache',
        'LOCATION': 'redis://redis:6379/1',
    }
}

# Tile caching
TILE_CACHE_TIMEOUT = 3600  # 1 hour
